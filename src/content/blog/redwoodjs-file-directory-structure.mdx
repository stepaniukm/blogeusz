One of the most debated topics in React development is file structure. Here's the approach I've settled on after years of working on both small and large codebases.

## The guiding principle

**Co-locate things that change together.**

If a component, its styles, its tests, and its types always change at the same time — they should live in the same directory.

## The structure

```
src/
├── components/
│   ├── Button/
│   │   ├── Button.tsx
│   │   ├── Button.test.tsx
│   │   └── index.ts
│   └── Card/
│       ├── Card.tsx
│       └── index.ts
├── pages/
│   ├── Home/
│   │   ├── Home.tsx
│   │   └── index.ts
│   └── Blog/
│       ├── Blog.tsx
│       └── index.ts
└── lib/
    ├── api.ts
    └── utils.ts
```

This scales beautifully — each feature is self-contained and can be moved, renamed, or deleted as a unit.
